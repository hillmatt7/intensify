# Security policy

## Supported versions

Only the most recent minor release receives security updates.

| Version | Supported |
|---|---|
| 0.1.x   | ✓ |
| < 0.1   | — |

## Reporting a vulnerability

Please **do not** open public GitHub issues for security problems.

Email the maintainer directly, or use GitHub's private security advisory
workflow:
<https://github.com/hillmatt7/intensify/security/advisories/new>.

Include:

- A description of the issue and its impact.
- A minimal reproducer.
- Affected versions and environment (Python, OS, active backend).

You can expect an initial acknowledgment within 7 days. Once the issue is
confirmed we will coordinate a release and public disclosure with you.
