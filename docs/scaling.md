# Scaling study

How does fit time scale with dataset size? This page answers that
directly for multivariate exp-Hawkes (M=5) against tick in decay-given
mode — the apples-to-apples setting.

All datasets were generated by the multivariate branching (Galton-Watson)
simulator, seeded with `42`, fit with β fixed to ground truth
(`tick` requires this; `intensify` opts in via `fit_decay=False`).
Runs are median of 3 timings after a warm-up fit to amortize JIT compile
in the intensify path. Reproduce:

```bash
# Generate scale datasets (one-time, ~1 min via branching sim)
.venv/bin/python benchmarks/reference_dataset.py \
    mv_exp_5d_scale_small mv_exp_5d_scale_medium \
    mv_exp_5d_scale_large mv_exp_5d_scale_xl mv_exp_5d_scale_xxl

# Run each library
.venv/bin/python benchmarks/run_scaling.py --lib intensify
micromamba run -n tickbench38 python benchmarks/run_scaling.py --lib tick
```

## Results

| N total | intensify time | intensify RMSE | tick time | tick RMSE | time ratio |
|---:|---:|---:|---:|---:|---:|
| 501 | 8 ms | 0.072 | 1 ms | 0.082 | 8× |
| 2,249 | 21 ms | 0.037 | 2 ms | 0.037 | 10.5× |
| 9,271 | 38 ms | 0.016 | 6 ms | 0.016 | 6.3× |
| 27,519 | 189 ms | 0.012 | 15 ms | 0.013 | 12.6× |
| 91,249 | 531 ms | 0.006 | 48 ms | 0.006 | 11× |

## What the curve says

**Both libraries scale linearly in N.** At the 91 k-event point,
intensify's per-event cost is ~5.8 μs; tick's is ~0.5 μs. The O(N·M)
recursive likelihood (`jax.lax.scan` with an M-vector state per source
dim) keeps the compute asymptotics correct.

**The ratio is stable at ~10×.** It does *not* grow with N. At small N
it's closer to 8× because of scipy's per-iteration Python dispatch
overhead; at large N that overhead is amortized by real compute and the
ratio settles to the constant factor between XLA on CPU and tick's
hand-tuned C++.

**Parameter-recovery RMSE is statistically indistinguishable.** By
N=9k both libraries recover true parameters to 3 significant figures
and from then on they track each other to within noise. That's the
asymptotic-consistency theorem doing its job — MLE converges to truth
regardless of which backend computes the likelihood.

## Practical implication

For realistic lab datasets (thousands to millions of events), both
libraries finish in well under a second per fit. Choose on
*capabilities* (kernel variety, joint-decay fits, Python ecosystem),
not on raw speed.

intensify's roadmap to close the constant factor (0.3.0 and later):

- Run L-BFGS-B inside a single `jax.jit` so each fit pays one XLA
  dispatch instead of one per iteration. Expected: 2–3× speedup on
  multivariate, reducing the gap to ~3–5× at large N.
- Per-source-dim recursion for joint decay fits (β varying across
  source dims). Removes the O(N²) fallback used in joint-fit mode.

Neither requires leaving Python.
